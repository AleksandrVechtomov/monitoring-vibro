## Установка:
1. Клонируйте репозиторий себе на компьютер
2. Откройте папку проекта в любой IDE. Убедитесь, что в терминале вы находитесь в папке проекта
3. Создайте виртуальное окружение  
```shell
python -m venv venv
```
4. Перейдите в виртуальное окружение и активируйте его
```shell
venv\Scripts\activate
``` 
5. Установите все необходимые библиотеки 
```shell
pip install -r requirements.txt
```

## Описание скриптов:
`file_watcher_daemon.py`  
Предназначен для автоматического отслеживания нового аудиофайла в указанной директории.   
Когда новый файл с расширением .wav появляется в этой директории (поддиректории тоже учитываются), 
скрипт извлекает из имени файла информацию о времени и моторе, 
затем вычисляет коэффициент здоровья мотора и сохраняет результаты в базе данных SQLite.  

`db_viewer.py`  
Этот скрипт подключается к базе данных audio_features.db, 
извлекает все записи из таблицы audio_features и выводит их в терминале.  

`audio_feature_extractor.py`  
Скрипт обрабатывает все .wav файлы в указанной директории и её подкаталогах, 
извлекает из имени файла информацию о времени и моторе, вычисляет коэффициент здоровья мотора, 
и сохраняет данные в базу данных audio_features.db.  

`db_data_fetcher.py`  
Скрипт извлекает данные о характеристиках мотора из базы данных в заданном временном интервале, 
фильтруя по имени мотора, и возвращает результаты в формате DataFrame. 
Затем строит график изменения признаков во времени, с отображением зоны возможных сбоев (failure zone).

## Описание базы данных

База данных `audio_features.db` используется для хранения данных о характеристиках работы моторов, 
извлечённых из аудиофайлов. 
Она организована в виде одной таблицы, которая содержит информацию о временных метках, моторах, 
путях к файлам и коэффициенте здоровья мотора.

### Структура базы данных

В базе данных есть одна таблица: `audio_features`

| Поле          | Тип данных   | Описание                                               |
|---------------|--------------|--------------------------------------------------------|
| `id`          | `INTEGER`    | Уникальный идентификатор записи (автоматически увеличиваемое значение). |
| `timestamp`   | `TEXT`       | Временная метка, когда был обработан файл (в формате `YYYY-MM-DD HH:MM:SS`). |
| `motor_name`  | `TEXT`       | Имя мотора, к которому относится аудиофайл.            |
| `file_path`   | `TEXT`       | Путь к аудиофайлу, в котором был извлечён признак.      |
| `feature_value` | `REAL`      | Извлечённый из аудиофайла признак, представляющий собой коэффициент здоровья мотора. |

### Пример записи

Каждая запись в таблице представляет собой информацию об одном аудиофайле, обработанном для извлечения признаков:

```plaintext
id | timestamp           | motor_name | file_path                | feature_value
---|---------------------|------------|--------------------------|---------------
1  | 2024-10-09 11:00:00 | m2         | data/m2_202410091100.wav | 0.5123
2  | 2024-10-09 11:05:00 | m2         | data/m2_202410091105.wav | 0.4872
3  | 2024-10-09 11:10:00 | m1         | data/m1_202410091110.wav | 0.3984
```

### Описание данных

- **`timestamp`**: Временная метка указывает, когда был записан аудиофайл, и соответствует дате и времени, закодированным в имени файла.
  
- **`motor_name`**: Каждое имя мотора (`motor_name`) соответствует уникальному идентификатору мотора, для которого были извлечены признаки.

- **`file_path`**: Путь к аудиофайлу, в котором был извлечён признак. Обычно файл имеет расширение `.wav` и имя, которое содержит информацию о моторе и временной метке.

- **`feature_value`**: Это числовое значение, извлечённое из аудиофайла, которое представляет собой коэффициент здоровья мотора.

### Примечание

Таблица будет автоматически создана при первом запуске программы, если она ещё не существует. 
Все данные о моторах, их характеристиках и аудиофайлах будут записаны в эту таблицу по мере обработки новых файлов.
